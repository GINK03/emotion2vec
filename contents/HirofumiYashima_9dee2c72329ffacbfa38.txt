\uff09\n\n\nWikipedia\uff08English\uff09 JSON-LD\n\n\nJSON-LD, or JavaScript Object Notation for Linked Data, is a method of transporting Linked Data using JSON. \nIt was a goal to require as little effort as possible from developers to transform their existing JSON to JSON-LD.\nThis allows data to be serialized in a way that is similar to traditional JSON.\nIt is a World Wide Web Consortium Recommendation that has been developed by the JSON for Linking Data Community Group before it has been transferred to the RDF Working Group[3] for review, improvement, and standardization.\nJSON-LD is designed around the concept of a \"context\" to provide additional mappings from JSON to an RDF model. The context links object properties in a JSON document to concepts in an ontology.\nIn order to map the JSON-LD syntax to RDF, JSON-LD allows values to be coerced to a specified type or to be tagged with a language. \nA context can be embedded directly in a JSON-LD document or put into a separate file and referenced from different documents (from traditional JSON documents via an HTTP Link header).\n\nExample\n{\n  \"@context\": {\n    \"name\": \"http://xmlns.com/foaf/0.1/name\",\n    \"homepage\": {\n      \"@id\": \"http://xmlns.com/foaf/0.1/workplaceHomepage\",\n      \"@type\": \"@id\"\n    },\n    \"Person\": \"http://xmlns.com/foaf/0.1/Person\"\n  },\n  \"@id\": \"http://me.example.com\",\n  \"@type\": \"Person\",\n  \"name\": \"John Smith\",\n  \"homepage\": \"http://www.example.com/\"\n}\n\n\n\n\nGoogle Developers \n\n\nJSON-LD\nJSON-LD is an easy-to-use JSON-based linked data format that defines the concept of context to specify the vocabulary for types and properties. \nGmail supports JSON-LD data embedded in HTML documents with the @context of schema.org, as in the following example:\n<script type=\"application/ld+json\">\n{\n  \"@context\": \"http://schema.org\",\n  \"@type\": \"Person\",\n  \"name\": \"John Doe\",\n  \"jobTitle\": \"Graduate research assistant\",\n  \"affiliation\": \"University of Dreams\",\n  \"additionalName\": \"Johnny\",\n  \"url\": \"http://www.example.com\",\n  \"address\": {\n    \"@type\": \"PostalAddress\",\n    \"streetAddress\": \"1234 Peach Drive\",\n    \"addressLocality\": \"Wonderland\",\n    \"addressRegion\": \"Georgia\"\n  }\n}\n</script>\n\n\n\nJSON for Linking Data\n\n\nThe JSON-LD Playground is a web-based JSON-LD viewer and debugger. If you are interested in learning JSON-LD, this tool will be of great help to you. Developers may also use the tool to debug, visualize, and share their JSON-LD markup.\n\n\nDigital Identity \uff09__\n\n#### __[Wikipedia\uff08English\uff09 _JSON-LD_](https://en.wikipedia.org/wiki/JSON-LD)__\n> __JSON-LD, or JavaScript Object Notation for Linked Data__, is a method of transporting Linked Data using JSON. \n>\n>It was a goal to require as little effort as possible from developers to transform their existing JSON to JSON-LD.\n>\n> This allows data to be serialized in a way that is similar to traditional JSON.\n>\n>It is a World Wide Web Consortium Recommendation that has been developed by the JSON for Linking Data Community Group before it has been transferred to the RDF Working Group[3] for review, improvement, and standardization.\n>\n>JSON-LD is designed around the concept of a \"context\" to provide additional mappings from JSON to an RDF model. The context links object properties in a JSON document to concepts in an ontology.\n>\n> In order to map the JSON-LD syntax to RDF, JSON-LD allows values to be coerced to a specified type or to be tagged with a language. \n>\n>A context can be embedded directly in a JSON-LD document or put into a separate file and referenced from different documents (from traditional JSON documents via an HTTP Link header).\n>\n>```{JSON-LD:Example}\n{\n  \"@context\": {\n    \"name\": \"http://xmlns.com/foaf/0.1/name\",\n    \"homepage\": {\n      \"@id\": \"http://xmlns.com/foaf/0.1/workplaceHomepage\",\n      \"@type\": \"@id\"\n    },\n    \"Person\": \"http://xmlns.com/foaf/0.1/Person\"\n  },\n  \"@id\": \"http://me.example.com\",\n  \"@type\": \"Person\",\n  \"name\": \"John Smith\",\n  \"homepage\": \"http://www.example.com/\"\n}\n>```\n\n#### __[Google Developers ](https://developers.google.com/schemas/formats/json-ld?hl=ja)__\n> __JSON-LD__\n>\n>JSON-LD is an easy-to-use JSON-based linked data format that defines the concept of context to specify the vocabulary for types and properties. \n>\n> __Gmail supports JSON-LD data embedded in HTML documents__ with the @context of schema.org, as in the following example:\n>\n>```\n<script type=\"application/ld+json\">\n{\n  \"@context\": \"http://schema.org\",\n  \"@type\": \"Person\",\n  \"name\": \"John Doe\",\n  \"jobTitle\": \"Graduate research assistant\",\n  \"affiliation\": \"University of Dreams\",\n  \"additionalName\": \"Johnny\",\n  \"url\": \"http://www.example.com\",\n  \"address\": {\n    \"@type\": \"PostalAddress\",\n    \"streetAddress\": \"1234 Peach Drive\",\n    \"addressLocality\": \"Wonderland\",\n    \"addressRegion\": \"Georgia\"\n  }\n}\n</script>\n>```\n\n#### __[JSON for Linking Data](http://json-ld.org/)__\n> The JSON-LD Playground is a web-based JSON-LD viewer and debugger. If you are interested in learning JSON-LD, this tool will be of great help to you.
