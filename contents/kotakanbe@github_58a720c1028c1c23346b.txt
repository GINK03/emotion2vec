Architecture\n\n![vuls-architecture.png](https://qiita-image-store.s3.amazonaws.com/0/22462/ff60362b-3164-f74d-408a-4d6114fde6f1.png)\n\n---\n\n# Features ([\u65e5\u672c\u8a9e](https://github.com/future-architect/vuls/blob/master/README.ja.md#abstract))\n\n- Scan for any vulnerabilities in Linux/FreeBSD Server\n    - Supports Ubuntu, Debian, CentOS, Amazon Linux, RHEL, FreeBSD\n    - Cloud, on-premise, Docker\n- Scan middleware that are not included in OS package management\n    - Scan N/W Devices, middleware, programming language libraries and framework for vulnerability\n    - Support software registered in [CPE](https://nvd.nist.gov/cpe.cfm) ([\u65e5\u672c\u8a9e](https://www.ipa.go.jp/security/vuln/CPE.html))\n- Agentless architecture\n    - User is required to only setup one machine that is connected to other target servers via SSH\n- Nondestructive testing\n- Pre-authorization is not necessary\u3000before scanning on AWS\n- Email and Slack notification are both available. (supports Japanese language) \n- Scan result is viewable on accessory software, TUI Viewer terminal or Web UI ([VulsRepo](https://github.com/usiusi360/vulsrepo)).\n\n---\n\n# Usage: Scan
